there are………..

  • sets of possible states!!! (as depicted by a set of variables and possibly some constraints and their truth tables)
  • topologies on those sets!! (state spaces, as depicted by the state machine diagrams, which show not only the states (as labeled circles) but also the ways u can get from each state to another (as labeled arrows))
  • topology templates!! which are equivalent to the state machine diagrams, but more compact and easier to manage, it depicts a topology in a more intuitive way in relation to the state variables

a topology template + a set of state variables that it depends on = some topology

the topology template is more useful for representing the dungeon itself in its physical form the expanded topology is more useful for representing the progression of the dungeon

  1. switch unpressed, platform recessed

  3. switch pressed, platform recessed
  4. switch pressed, platform risen

  1. switch pressed and platform risen
  2. switch unpressed and platform recessed

without the constraint u get this truth table S = switch pressed P = platform risen

with the “S if and only if P” constraint, u get this truth table which rly shows that they r exactly the same thing in this case so u can simplify it to just one variable, say S
